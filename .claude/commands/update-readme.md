README.md를 현재 프로젝트 상태에 맞게 전체 업데이트한다.

## 언제 사용하나

`scripts/sync-readme.sh`는 스킬 목록과 훅 목록만 기계적으로 갱신한다.
아래와 같은 변화가 있을 때는 이 스킬을 직접 호출해 전체 내용을 재작성한다:

- CLAUDE.md 컨벤션이 크게 변경됐을 때
- 새 패키지/기술 스택이 추가됐을 때
- 프로젝트 목적·구조가 바뀌었을 때

## 절차

1. 다음 파일을 **병렬**로 읽는다:
   - `CLAUDE.md` — 컨벤션 및 아키텍처
   - `package.json` — 의존성 및 스크립트
   - `.claude/commands/` 내 모든 `.md` 파일 — 스킬 목록과 설명
   - `.claude/settings.json` — 훅 설정
   - `src/` 폴더 구조 (`find src -type d`)

2. README.md의 각 섹션을 현재 상태에 맞게 업데이트한다:
   - **기술 스택 표** — package.json 기준으로 갱신
   - **프로젝트 구조** — 실제 src/ 구조 반영
   - **슬래시 커맨드 표** — .claude/commands/ 기준으로 갱신 (AUTO 마커 섹션)
   - **훅 표** — .claude/settings.json 기준으로 갱신 (AUTO 마커 섹션)
   - **코딩 컨벤션 요약** — CLAUDE.md에서 핵심만 발췌

3. 모든 내용은 한국어로 작성한다. 기술 용어(컴포넌트명, 라이브러리명)는 영어 유지.

4. 파일을 저장한 후 변경된 내용을 요약해서 보고한다.
